# Instrumentation Minimum v0.2

Status: active QA / Stage 4 shadow-mode operating artifact.

Source: Consigliere Core Spec v0.5.1, section 10, adapted to current repo structure.

## Purpose

ACI-OS should learn from use without collecting secrets.

Instrumentation is not surveillance and not engagement optimization. It exists to improve product judgment, routing, question quality, escalation precision, and failure detection.

## Stage 4 Operating Rule

Stage 4 journal use is live from 2026-07-04 for Tom's internal use only.

Use pattern form only: no real facts, no names, no company secrets, no client data, no whistleblower identities, no privileged material.

Until Gate 2 passes, tool outputs are shadow mode. They are drafts to test, not advice to rely on. Every override, disagreement, or delta should be recorded because those deltas are the product learning.

## Monthly Stats

Track at least:

- Program element frequency.
- Mode distribution, including Mirror mode.
- Depth distribution.
- Time-of-day distribution for the 22:30 thesis.
- Device / channel split, including future voice.
- Triage asked versus skipped.
- "Something else" rate in the situation picker.
- Color distribution, including earned routine green rate.
- Escalation precision: correct, missed, unnecessary, unclear.

## Question-Bank Analytics

For each front-door question:

- Asked.
- Answered.
- Skipped.
- "Don't know."
- Changed the recommendation: yes / no.
- Should be kept, revised, or removed.

Questions that never change the recommendation should be cut. Gaps that judgment later needed should become new questions.

## Per-Output QA Checklist

For non-trivial outputs, check:

- Role label present.
- Legal / Compliance / management boundary evaluated where relevant.
- Color rules respected if color is used.
- Epistemic content proportional to stakes.
- As-of date where time-sensitive.
- Program element tag where useful.
- No clearance language.
- Escalation evaluated.
- Speed-1 closes doors only in crisis mode.
- Mirror mode answers both the procedural question and the person / position question.
- Single next action appears last where stakes are high.

## Boundary

Do not use instrumentation for streaks, engagement hooks, gamification, behavioral retention, or pressure to keep the user in the product.
