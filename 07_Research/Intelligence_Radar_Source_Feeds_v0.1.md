# Intelligence Radar Source Feeds v0.1

Status: active internal source-feed checklist for the ACI-OS intelligence radar.

Purpose: give the weekly ACI-OS Intelligence Radar a stable starting list of official and serious research sources, while preserving source discipline. This file is a search and monitoring aid, not authority by itself.

Stage status: internal only. Stage 0b blocks publication, testers, pilot links, commercial use, or external distribution of radar outputs.

## Use Rule

- Use this file to start each weekly radar scan.
- Cite a link only if it is already verified in `07_Research/Source_Register_v0.1.md` or verified during the report run.
- Treat official sources as primary, research sources as secondary, and press/commentary as weak signal only.
- Do not add any source output to methodology, expert routes, prompts, test cases, or knowledge bundles without an ACT / PARK / DISCARD verdict and Tom's explicit YES token.
- If a feed produces no material signal, say that briefly; do not invent significance from silence.

## Weekly Scan Steps

1. Start with the standing verified feeds.
2. Check official feeds to verify for material current-week developments.
3. Search research feeds for compliance-program and human-interaction signals.
4. Separate official, research, and commentary sources.
5. Map each substantive item to the 7-element backbone and cross-cutting tags.
6. Produce ACT / PARK / DISCARD recommendations.
7. Add decision cards for any proposed documentation, knowledge, methodology, response-behavior, expert-route, or test-case change.
