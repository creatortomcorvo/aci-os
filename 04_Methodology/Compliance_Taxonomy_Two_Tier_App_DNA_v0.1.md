## Purpose

This taxonomy defines the compliance topic universe for ACI-OS.

It prevents sanctions from becoming the default lens, makes corporate governance and ESG explicit, and gives the decision journal a stable tagging system so future routes and agents are selected from observed use rather than preference.

## Core Rule

ACI-OS uses a two-tier compliance taxonomy:

- Tier 1 is the universal compliance core.
- Tier 2 is the sector calibration layer.

The taxonomy does not replace the 7-element backbone. It sits inside the backbone as the standards, controls, risks, examples, source routes, and calibration vocabulary used by the product.

Tier 1 usually maps most visibly to Element 3 Standards & Controls, but the same topic can also trigger governance, risk assessment, training, speak-up, monitoring, or remediation work across Elements 1-7.

## Product Behavior Rules

1. Start from the user's facts and pressure pattern, not from a favorite topic.
2. Select response mode before expert route.
3. Use Tier 1 to identify the universal compliance family.
4. Use Tier 2 only when the industry context changes examples, sources, thresholds, controls, or escalation.
5. Keep sector packs as calibration layers unless journal data and gate discipline justify a new specialist route.
6. Do not copy external codes into ACI-OS. Use them to benchmark coverage, sharpen examples, and verify source-backed claims.
7. Treat AI as an emerging universal compliance topic in Tier 1 and as a sector pack in Tier 2 when the user is building or governing AI systems.

## Decision Journal Rule

Every non-trivial Stage 4 journal entry should include:

- Tier 1 tag: one primary family, with secondary tags if useful.
- Tier 2 tag: only if a sector pack materially shaped the answer.
- Program element tag: the 7-element backbone mapping.
- Follow-up signal: article, test case, methodology fix, route candidate, or none.

This creates the compliance coverage dashboard.

## Agent Selection Rule

Agent #2 is not selected because a topic is important in theory.

Agent #2 is selected only when Stage 4 journal frequency, failure patterns, user need, source readiness, and gate discipline show that a new specialist route is justified.
