# Compliance Template Pack Index v0.1

Status: active planning index.

Source: founder note on durable compliance-document architecture, reviewed 2026-07-04.

Purpose: define the recommended build order for future ACI-OS compliance-document templates.

This file is an index, not the finished template pack.

## Build Order

| Order | Template | Why first / why later |
| ---: | --- | --- |
| 1 | Document Governance Standard | Controls the whole library: taxonomy, ownership, approval, review, versioning, access, retention, retirement. |
| 2 | Code of Conduct | Visible foundation of the compliance program. Should be drafted after hierarchy and governance are clear. |
| 3 | Compliance Management Policy / Program Charter | Defines governance, mandate, roles, escalation, reporting, and operating model. |
| 4 | Topic Policy Template | Reusable for anti-bribery, conflicts, sanctions, competition, harassment, speak-up, investigations, data, records. |
| 5 | Procedure Template | Reusable for approval workflows, due diligence, screening, case intake, investigations, reporting, legal hold. |
| 6 | Work Instruction / SOP Template | Reusable for system-level task instructions. |
| 7 | Guideline / FAQ / Playbook Template | Reusable for examples, judgment calls, red flags, and business responses. |
| 8 | Register / Dashboard Template | Reusable for evidence, monitoring, exceptions, approvals, and management reporting. |
| 9 | Local Addendum Template | Reusable for country-specific mandatory-law modifications. |
| 10 | Review and Gap-Assessment Checklist | Reusable for annual review, audit preparation, and remediation planning. |

## Minimum Sections By Type

### Code of Conduct

- CEO / Board message.
- Scope.
- Values.
- Decision test.
- Main risk chapters.
- Speak Up.
- Non-retaliation.
- Consequences.
- Where to find more.
- Owner and review cycle.

### Policy

- Purpose.
- Scope.
- Core rules.
- Definitions.
- Roles.
- Approval / escalation.
- Records.
- Exceptions.
- Consequences.
- Related procedures.
- Review cycle.

### Procedure

- Trigger.
- Inputs.
- Steps.
- Decision points.
- Roles.
- Evidence.
- Timelines.
- Escalation.
- Controls.
- Exceptions.
- Outputs.

### Process

- Start / end.
- Stakeholders.
- Swimlanes.
- Handoffs.
- Systems.
- Control points.
- KPIs.
- Failure modes.

### Guideline

- Link to policy.
- Examples.
- Red flags.
- FAQs.
- Good / bad scenarios.
- Escalation points.

### Work Instruction / SOP

- Task.
- User role.
- Exact steps.
- Screens / fields where relevant.
- Records.
- Common errors.
- Help contact.

### Local Addendum

- Country / entity.
- Global policy reference.
- Local Legal owner.
- Local Compliance owner.
- Stricter local rules.
- Prohibited local conduct.
- Local thresholds.
- Works council / employee notice requirements where relevant.
- Local reporting channels.
- Local record retention.
- Local approval deviations.
- Effective date and review cycle.
