# ACI-OS

ACI-OS is the permanent local knowledge base for the Compliance Consigliere project.

This repository contains the project foundation, expert maps, test cases, article pipeline, research notes, and product logic for the first Chief Consigliere prototype.

The long-term product idea is not to replace due diligence providers, sanctions databases, speak-up tools, training platforms, or other compliance IT systems. ACI-OS should become the intelligent interface between the compliance officer, the compliance program, the business, external providers, and company systems.

## Core Repository Areas

- `00_Project` - project memory, discipline, history, current sprint, and handoff logic.
- `04_Methodology` - response modes, front-door design, moment discovery, response patterns, and relationship lifecycle logic.
- `05_Experts` - source-backed expert routes, starting with sanctions.
- `06_Testing` - evaluation checklists, raw-vs-Consigliere tests, and failure taxonomy.
- `10_Product` - product principles, prompts, strategy, memory, calibration, and governance notes.
- `12_Document_Architecture` - compliance-document operating-system architecture: Code, policies, procedures, work instructions, guidelines, forms, registers, local addenda, lifecycle, evidence, and template-pack planning.
- `99_Parked` - useful material that is not yet active or not yet validated.
